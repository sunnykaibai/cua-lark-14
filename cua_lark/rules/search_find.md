# Search And Find

Use:
- For 找到/搜索/查找/最近一条包含/contains, use the current product's visible search/find entry before scroll-scanning history.
- Match the search surface to the follow-up operation: message search for message ops, picker search for chat object send/share, document/product search for opening objects.
- If a suitable search field is visible and the query is known, batch stable focus/type/enter.
- After typing a query into a search field, inspect the next screenshot. If the visible query did not change, do not repeat the same type action; refocus/clear the field once or use Enter/visible result if the intended query is already present.
- Use sender/user filters only when the instruction explicitly says sent by/from that person.
- After a result jump, operate the highlighted/selected live target in the main content area.
- If a side search panel remains but the live target is usable, keep focus on the live target; close the panel only if it blocks controls.
- If the same result/jump was already clicked and nothing changed, choose another visible route instead of repeating.

Avoid:
- Treating manual scrolling or visible historical bubbles as the first strategy for search/find tasks.
- Finishing from a result preview when the user requested an operation on the live object.
- Re-running the same search when a useful result state is visible.
- Repeating the same type action into a search field when the visible query did not change.
- Applying sender filters only because the conversation has that person's name.
- Using global/product search as a substitute for a chat share picker when the goal is to send/share into chat.

Batch:
- Batch visible stable search-field actions; stop before acting inside a newly opened result page, picker, popup, or panel.
