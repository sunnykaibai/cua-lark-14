# Attachment And Share

Use:
- For chat rich-object send/share, prefer the chat composer attachment/share button; search after the object picker opens.
- For local file/image pickers, select the requested file object and confirm with the visible Open/Choose button.
- If a confirmation/composer preview appears, send that real object from the visible UI.
- For cloud docs sent as chat messages, stay in IM/picker flow; document-product permission sharing is a different task.
- Global/product search results can locate/open an object, but are not a chat send surface unless a visible send/share control exists.
- In a share/send popup, search/select the intended recipient, then use the visible send/share button.
- For personal-card tasks, progress through visible plus/more options, card/contact picker, selected card/contact, and send as a real card.

Avoid:
- Send a file path, document title, contact name, or explanation as a plain text substitute.
- Use `command v` to paste an invisible share link.
- Invent row controls that are not visible.
- Repeatedly click a search result row that only highlights/previews and exposes no send/share route.
- Treat a document permission/invite dialog as completion for sending a cloud-document card into a chat.

Batch:
- Batch visible stable select/confirm/send controls; stop before acting inside a not-yet-visible picker/dialog/popup.
