# Rich Text

Use:
- Rich-text tasks require visible formatting state or a formatted outgoing message.
- Use visible formatting controls, such as bold B, before typing when formatting should affect new text.
- Type requested content once after the intended formatting state is active.
- If the formatted message is sent but a duplicate draft remains, clear the draft before finishing.

Avoid:
- Formatting via `command a` unless the intended composer text is visibly selected.
- Re-typing the same rich text when the newest live message already satisfies the task.
- Treating plain unformatted text as rich text.

Batch:
- Batch stable formatting, typing, and send actions while later targets remain visible and unchanged.
